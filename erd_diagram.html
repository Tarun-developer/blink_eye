Database Architecture Overview

Primary Keys: Unique identifiers for each record
Foreign Keys: References to related tables (tenant_id for multi-tenancy)
Junction Tables: Many-to-many relationships (user_roles, role_permissions)
Global Tables: Shared across all tenants (permissions, content_master)

Key Features:

  • Multi-Tenant Architecture: Each hospital operates in complete isolation
  • Scalable Design: Supports unlimited hospitals with shared infrastructure
  • Healthcare Compliance: Secure patient data management
  • Role-Based Access: Granular permissions for different user types
  • Comprehensive Tracking: Full audit trail of all operations
